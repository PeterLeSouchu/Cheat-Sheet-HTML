HTML / CSS Cheat Sheet

Table

< table > → create a table

< thead / tbody / tfoot > → table structure (thead and tfoot have a weighter font than main)

< tr > → table row, one row always begin with < tr >

< th > → cell data usualy use in thead, but we can also use it in tbody or tfoot in order to make a cell data more important, and it's nested in < tr >

< td > → cell data nested in < tr >

< caption > → caption of table, can be placed at first/the end/

< colspan / rowspan > → merge column or row, use as an attribut in < td >

thead th th th th
tbody td td td td with rowspan="2"
tbody td with colspan="2" td
tfoot td td td td
Here,we can find the caption of this table

Form

< form > → create a form

< input > → create input (text / number / password / date / search... )

< label > → create a text which is link with input (("for" in label) = ("id" in input)). Can be placed before / between / after input.

type="radio" :

type="checkbox"

Bonus

Put a icon web site → link rel="shortcut icon" href="./image.png"

< section/div > → section is a semantic tag unlike div tag

< article > → use this tag instead section tag when we talk about a independent part of the site ('title+p+img' for example)

< aside > → use this tag when you want to give additional information to your section/article

reset CSS → use this to reset CSS. Place above other stylesheet or this will overwrite other style

Flexbox

Pseudo-class

hover → when the client hover an element.

active→ when the client activate an element, usualy when he clicks on it.

focus→ when an element receive the "focus", usualy when the client is in inputs.

visited → when an element is visited, usualy "links".

placeholder → modify the placeholder of inputs.

select → modify appearence of an selected element, usualy use on < p >.

after / before → create an child element at the beginning / end of parent element. With this one, we have to write "content" property.

Grid

Display : grid ; → change display "block" to "grid".

grid-template-columns : / grid-template-rows : repeat ( number, value ) ; | value value value ;

⤷ either we use repeat method or we use value method and we write values as many times as we want columns / rows. With the last one we can use differents values.

grid-template-areas : 

            "name1   name1   name2"
            "name1   name1   name2"
            "name3   name3   name3" ;

⤷ with this property we can build our grid, by assigning names to each child. In this example our grid has 3 colums and 3 rows.

grid-areas : name1/name2/name3;

⤷ with this property we name our child. This one has to be placed in the chlid container instead of the 3 others.
